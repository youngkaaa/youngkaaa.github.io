GLES&EGL-GLES-01

7月 20 2022

前面 GLES&EGL-EGL-01 、 GLES&EGL-EGL-02 、 GLES&EGL-EGL-03 中讲了 EGL 的相关内容， 趁热打铁，接下来介绍 OpenGL ES ，也就是前面一直提到的一系列 glXXX() 方法。我们直到，这一系列的 gl 方法的作用是用来绘制的，而前面的 EGL 为其准备好了一个绘制环境，比如创建好一些 context 实例，创建好了“画板”等，这些准备好了之后，就该我们的 gl 方法上场了。如果把 gl 比作是歌手（比如周杰伦，嘿嘿）上台演出的话，那么 EGL 就是工作人员，幕后人员，他们为歌手搭好舞台，调试好设备，歌手只需要上台唱歌就行了，歌手唱完歌，他们再负责收拾干净场地就行。

同时，在后续的分析中，我们看的都是gl函数在 libagl 中的函数实现。所以后续提到的类，就都是位于 libagl 中的，没有之前分析 EGL 时一直区分的 agl侧，egl侧了。

glGenTextures

glGenTextures() 方法用来生成一个纹理。

glGenTextures() 方法接收两个参数：

① GLsizei n ：它是一个GLsizei 类型的变量，而 GLsizei 的 定义是：typedef int GLsizei; ，也就是说后续可以把它当做是int。而它在这里表示的是要生成的纹理个数。

② GLuint *textures ：它是一个GLuint 类型的变量，而GLuint 的 定义是：typedef unsigned int GLuint; ，也就是说后续可以把它当做是unsigned int 。而它在这里表示的是要生成的纹理存储地址，而前面的参数 n 表示它的长度。简单说可以把他俩当做是数组和数组长度。

接着看它内部的逻辑。首先会调用 ogles_context_t::get() 方法来获取当前线程中之前保存的 ogles_context_t 实例。而这个实例是之前什么时候保存的呢？还记得前面将 EGL 时提到的 eglMakeCurrent() 方法嘛？不记得的话可以翻看 GLES&EGL-EGL-03 再复习下。在这个方法会最终调用到 libagl 中的 eglMakeCurrent() 方法，进而执行到 makeCurrent() 方法内，在该方法中会将新传入进来的 EGLContext ，也就是 ogles_context_t 实例通过 setGlThreadSpecific() 方法保存起来，保存到该线程中。

那么再此之后，执行到这里的 glGenTextures() 方法中就可以取出之前保存的那个 ogles_context_t 实例了。接着调用其内部 surfaceManager->getToken() 方法来完成纹理id的生成。

可以看到，这里的 EGLSurfaceManager 是继承自 TokenManager 的，然后在其基础上额外扩充了 mTextures 属性，该属性类型可以理解为是一个 Map ，key 是 GLuint ，也就是个int（其实是 unsigned int ，图方便后续都将 GLuint 统一当做 int 来理解），value 是其对应的一个 EGLTextureObject 实例。

glBindTexture

在正式看 glBindTexture() 之前，先插入一段对于 ogles_context_t.textures 初始化的分析，这样有助于后续的分析。

ogles_context_t.textures 初始化

ogles_context_t.textures 的初始化，同样是在之前的 eglCreateContext() 中，在创建并初始化 ogles_context_t 实例时，会执行到 ogles_init() 方法中，此时会完成该属性的赋值初始化：

在 ogles_init() 方法中，实际会调用 ogles_init_texture() 方法来初始化该 ogles_context_t->textures 中的属性，在看实际的ogles_init_texture() 方法之前，先熟悉下即将要遇到的这些新类型：

ogles_context_t->textures 的类型是 texture_state_t ，它内部有好几个属性，其中包括一个长度为 GGL_TEXTURE_UNIT_COUNT = 2 的 texture_unit_t 数组，名字叫做 tmu ，这个名字后面会经常遇到的。而 texture_unit_t 中只定义了三个属性，表示一个纹理相关的整体，比如表示纹理id的 name，纹理对象的 texture 属性。

接下来再来看 ogles_init_texture() 方法是怎么初始化 ogles_context_t->textures 的，也就是怎么初始化 texture_state_t 的。

首先就是将其 packAlignment 和 unpackAlignment 属性都初始化成 4，接着新建一个 EGLTextureObject 实例，来作为默认的纹理对象保存给其 defaultTexture 属性。

最后调用 bindTextureTmu() 方法，将这个刚创建出来的默认 EGLTextureObject 实例传入，来完成 tum 数组的初始化。

外部调用 bindTextureTmu() 方法时，传入的第二个参数是变化的，即循环的索引从0变化到1，而其他的三个参数都是固定的。

而 bindTextureTmu() 方法中，会将入参 tum 作为索引，找到 texture_state_t.tum 数组中对应的 texture_unit_t 实例，将入参 texture 和 tex 分别更新到该 texture_unit_t 实例中的 name 和texture 属性中去，而这里这俩参数都是固定的，即分别是 0 和 新创建的默认 EGLTextureObject实例。

最后会调用 invalidate_texture() 方法来将 texture_unit_t 实例中另外一个 flag 属性初始化成 0xFF。

所以循环完毕之后，exture_state_t.tum 数组中的texture_unit_t 实例中的 name 都是0， texture都是同一个刚创建出来的默认的EGLTextureObject 实例，flag 都是 0xFF 。

所以说，eglCreateContext() 方法执行完毕之后，其对应的ogles_context_t.textures 中就已经被初始化了，包括其中的默认纹理对象 defaultTexture 。

glTexParameteri

先看源码：

glTexParameteri() 方法就比较简单了，先调用 c->textures.tmu[c->textures.active].texture 来获取之前调用 glBindTexture() 方法时保存的 EGLTextureObject 实例，这也就是为什么在这个方法的参数中，没有了前面的纹理id，因为前面调用过 glBindTexture() 方法将对应的纹理id的相关信息绑定保存了起来，后续只需要去拿就行了。这也正是 glBindTexture() 方法的作用。

而拿到这个 EGLTextureObject 实例之后，会判断入参 pname ，全称时 paramName ，也就是参数名，当然对应的还有参数值 param 。这里根据参数名来执行不同的保存逻辑，下面分类看下。

① pname = GL_TEXTURE_WRAP_S ：则此时会参数值保存到 textureObject->wraps 中。

② pname = GL_TEXTURE_WRAP_T：则此时会参数值保存到 textureObject->wrapt 中。

③ pname = GL_TEXTURE_MIN_FILTER：则此时会参数值保存到 textureObject->min_filter 中。

④ pname = GL_TEXTURE_MAG_FILTER：则此时会参数值保存到 textureObject->mag_filter 中。

⑤ pname = GL_GENERATE_MIPMAP ：则此时会参数值保存到 textureObject->generate_mipmap 中。

关于这些参数代表的含义，详见：https://zhuanlan.zhihu.com/p/103881133

根据参数名保存好参数值之后，会调用 invalidate_texture() 方法来将该 EGLTextureObject 中的 flag 置为 0xFF ，表示其内部的属性发生了变化。
